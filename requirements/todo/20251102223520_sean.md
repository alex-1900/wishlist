# 需求列表
- 重构 http 路由

# 需求详情
## http 路由原则
- 路由的 request method 只使用 `GET` 和 `POST`
- 查询操作尽量使用 `GET` 请求
- 写入操作尽量使用 `POST` 请求
- **不需要**遵循 `RESTFUL` 原则，url 中不同单词用中横线 `-` 分隔，尽量语义化

## 重构 module
`src/module/` 目录中，应该按照业务划分为不同的模块（但现在只有单一的业务层级）。

建立 Account 模块（`src/module/account`），

将 `src/module/` 下原有的代码迁移到 `src/module/account` 目录下
